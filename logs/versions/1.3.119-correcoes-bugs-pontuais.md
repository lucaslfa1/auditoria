# v1.3.119 — Correções de bugs pontuais (handover)

**Data:** 2026-06-11
**Tipo:** Bugs — robustez de infraestrutura
**Custo:** deploy backend
**Branch:** `handover`

## Correções

### BUG-001 — `except: pass` no parse do retry-after 429 do Whisper
`backend/transcription_providers/azure.py`: o bloco que extrai o tempo de
retry da resposta 429 engolia QUALQUER exceção sem rastro. Agora
`except Exception` + `logger.debug(exc_info=True)` — comportamento idêntico
(backoff default 10s), com rastro para debug de rate-limit.

### SQL com f-string em `repositories/admin_criteria.py`
`_safe_select_rows` montava `SELECT {cols} FROM {table}` por f-string.
Os nomes já vinham de whitelist interna + `information_schema` (não era
injeção explorável), mas agora usa `psycopg2.sql.SQL/Identifier` — quoting
formal, defesa em profundidade contra caller futuro com input externo.

### Vazamento de conexão quando `putconn` falha
`backend/db/connection.py` (`PooledConnectionContext.close`): se
`_db_pool.putconn` lançasse (pool fechado durante shutdown, re-init), a
conexão nativa ficava aberta. Agora o guard fecha `conn.close()` direto e o
`finally` continua liberando o slot do semáforo.

### Lock do ciclo de automação com conexão stale
`backend/core/automation_engine.py` (`_AutomationCycleLock`):
- `refresh()`: erro de conexão/commit era exceção CRUA no meio do ciclo;
  agora vira `False` (lock perdido) → o ciclo aborta limpo via
  `AutomationLockLostError`, sem stacktrace de psycopg2 solto.
- `release()`: ganhou UMA re-tentativa com conexão NOVA antes de desistir
  para o TTL (30 min). Falha dupla loga e deixa o TTL assumir (comportamento
  anterior), mas a maioria das falhas transitórias agora libera o lock na
  hora — sem atrasar o ciclo seguinte.
