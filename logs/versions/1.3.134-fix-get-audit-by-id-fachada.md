# v1.3.134 — Fix do bug latente: fachada `get_audit_by_id` ausente

**Data:** 2026-06-12
**Tipo:** Bugfix (latente, pré-existente) + teste de regressão
**Custo:** nenhum
**Branch:** `main`

## O bug (encontrado na extração v1.3.133, anterior a ela)

`get_audit_by_id` era chamado em dois pontos SEM existir no escopo:

1. `db/audit_media.py` → `recover_audit_audio_from_classified_queue`, no
   branch defensivo `audit = audit or dbm.get_audit_by_id(audit_id)` —
   o atributo não existia na fachada `db.database` (AttributeError se
   executado; todos os callers atuais passam `audit` preenchido).
2. `db/database.py` → `finalize_contestation_review` (nome livre →
   NameError incondicional se chamada).

## Por que nunca explodiu

- O endpoint vivo de veredito (`routers/review.py`) chama o REPOSITORY
  direto, não a fachada.
- Os testes que exercitam `database.finalize_contestation_review`
  (`test_review_module_flow.py`, `test_audit_flow_fixes.py`) estão TODOS
  com skip desde a migração PostgreSQL ("legacy DB_NAME pattern") — a
  suite verde mascarava o NameError.

## Correção (opção A — fachada, não remoção)

Adicionada a fachada `get_audit_by_id` em `db/database.py` delegando para
`repositories.audits.get_audit_by_id` (mesmo padrão das irmãs). Era a única
leitura de audits sem fachada. Conserta os dois call sites de uma vez,
mantém `finalize_contestation_review` funcional (é a versão mais completa
do fluxo: rebalanceia a fila do operador e re-espelha o Arquivo Salvo) e
destrava os testes legados se forem reativados.

## Teste de regressão

`test_audit_audio_recovery.py::test_recovery_without_audit_param_resolves_audit_via_facade`
— exercita o branch antes morto (`audit=None`) e prova que a resolução via
fachada funciona e respeita monkeypatch.

## Observações registradas (sem mudança de comportamento)

- Os 13 testes de contestação seguem skipped ("legacy DB_NAME pattern") —
  candidatos a modernização futura.
- O endpoint vivo de veredito NÃO rebalanceia a fila nem re-espelha o
  Arquivo Salvo (diferente da fachada). Pode ser intencional ou gap —
  avaliar separadamente antes de mexer.
