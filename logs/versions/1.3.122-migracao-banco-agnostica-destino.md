# v1.3.122 — Pacote de migração do banco agnóstico de destino (decisão do Lucas)

**Data:** 2026-06-11
**Tipo:** Handover — ajuste de escopo
**Custo:** nenhum (docs e scripts; nada roda em produção)
**Branch:** `handover`

## Contexto

Decisões de escopo comunicadas pelo Lucas durante o handover:
1. **Nada será migrado nem provisionado por este lado** — nenhum serviço
   Azure será criado. Entregamos scripts/docs/código dormente; quem executa a
   migração e o deploy é o time de engenharia da empresa, na infra deles.
2. **O serviço de banco do destino NÃO está definido** — pode não ser o Azure
   Database for PostgreSQL. O pacote de migração (v1.3.121) assumia Azure.

## Mudanças

- `docs/10-migracao-azure-db.md` → **`docs/10-migracao-banco.md`**: reescrito
  agnóstico de destino. Nova seção "Requisitos do destino (inegociáveis sem
  reescrita)": engine PostgreSQL (psycopg2 em todo o backend), versão >= 17,
  pgvector, TLS. Azure Flexible Server aparece apenas como EXEMPLO anotado
  (junto de RDS/Cloud SQL/auto-hospedado).
- `scripts/migration/02_restore_azure.{sh,ps1}` →
  **`02_restore_destino.{sh,ps1}`**; env `AZURE_DATABASE_URL` →
  **`TARGET_DATABASE_URL`** (alinhada com o `03_validate.py`, que já usava
  SOURCE/TARGET).
- Referências cruzadas atualizadas (01_dump aponta para o novo nome).

## Sem mudança de comportamento
`03_validate.py` e `01_dump_neon.*` intocados na lógica. O adapter
`azure_blob` de media storage (em andamento) permanece como código DORMENTE
atrás de `MEDIA_STORAGE_BACKEND` — só ativa se o time optar por Blob Storage.
