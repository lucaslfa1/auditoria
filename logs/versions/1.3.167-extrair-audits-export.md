## O que foi feito

Extraído de `backend/repositories/audits.py` (1584 linhas) o cluster read-only de
**export para fechamento/BI e listagens** para o módulo irmão
`backend/repositories/audits_export.py` (~295 linhas movidas).

Símbolos movidos (3):
- `get_audits_for_export` — monta o dict consumido pelo FECHAMENTO/BI (chaves e
  semântica são CONTRATO; isolá-las num arquivo próprio facilita o handover)
- `list_pending_dispatch_audits` — auditorias paradas em `awaiting_pair`
- `_resolve_audio_hash_fallback` — resolve metadata de áudio por `input_hash`
  (movido junto para manter o novo módulo como folha pura, sem import de volta)

## Compatibilidade

Os 3 nomes são reexportados de `repositories.audits`
(`from repositories.audits_export import (...)  # noqa: E402,F401`). `get_audit_by_id`
(que fica em audits.py) usa `_resolve_audio_hash_fallback` via o nome reexportado;
a fachada `db.database` e `routers/review` acessam `get_audits_for_export`/
`list_pending_dispatch_audits` como atributo de módulo (os patches de teste miram
`routers.review.audits.get_audits_for_export` / `main.database.get_audits_for_export`
— preservados pelo reexport).
