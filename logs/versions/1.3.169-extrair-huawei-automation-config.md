## O que foi feito

Extraído de `backend/core/huawei_sync.py` (1461 linhas) o cluster de
**configuração / credenciais / tuning de runtime** do sync para o módulo
`backend/core/huawei/automation_config.py` (~185 linhas movidas).

Símbolos movidos (18 = 6 constantes + 12 funções):
- Constantes: `DEFAULT_HUAWEI_SYNC_DOWNLOAD_LIMIT`,
  `DEFAULT_HUAWEI_SYNC_MIN_DURATION_SECONDS`,
  `DEFAULT_HUAWEI_SYNC_MAX_DURATION_SECONDS`,
  `DEFAULT_HUAWEI_SYNC_DOWNLOAD_CONCURRENCY`,
  `DEFAULT_HUAWEI_SYNC_CLASSIFY_CONCURRENCY`,
  `DEFAULT_HUAWEI_AUTO_AUDIT_CONFIDENCE_THRESHOLD`
- Credenciais/flags: `_ensure_enabled`, `_load_config`, `_missing_credentials`,
  `_env_flag`, `_should_run_auto_classification_after_sync`
- Tuning: `_coerce_int`, `_coerce_float`, `_runtime_int_config`,
  `_runtime_float_config`, `_effective_download_attempt_limit`,
  `_get_huawei_auto_audit_confidence_threshold`, `_get_duration_limits_for_sector`

Funções puras (leem `configuracoes`/env e devolvem escalares) — não tocam cliente
Huawei/OBS, fila nem o estado do orquestrador (`contadores`).

## Compatibilidade

Os 18 nomes são reexportados de `core.huawei_sync`
(`from core.huawei.automation_config import (...)  # noqa: E402,F401`). Resolvem:
callers internos do orquestrador, `core/huawei/sync_classification.py` (via
`hs._env_flag`/`hs._get_huawei_auto_audit_confidence_threshold`),
`core/huawei_d_minus_1.py` e ~10 scripts que fazem
`from core.huawei_sync import _load_config`. Os patches de teste
(`patch.object(huawei_sync, '_load_config'/'_ensure_enabled')`,
`patch.object(huawei_sync.database, 'get_config_value')`,
`huawei_sync.DEFAULT_HUAWEI_SYNC_DOWNLOAD_LIMIT`) seguem válidos via reexport.

⚠️ `_env_flag` mantém a semântica Huawei (`Optional[bool]`, default `None`) — NÃO
confundir com `core.config._env_flag` (`bool`, default `False`). Verificado em
runtime que `huawei_sync._env_flag` resolve a versão Huawei.
