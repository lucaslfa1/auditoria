- cache hit
- transcrição concluída

- reauditoria concluída

    build_internal_quality_trace,

## Patch proposto para `backend/core/evaluation.py`

### Objetivo
Criar telemetria interna do reparo de JSON inválido sem alterar o contrato público do resultado.

### Helper a adicionar
```python
def _log_json_repair_event(
    *,
    event: str,
    attempt: int,
    provider: str,
    schema_hint: str,
    text: Optional[str],
    error: Optional[Exception] = None,
) -> None:
    payload = {
        "event": event,
        "attempt": attempt,
        "provider": provider,
        "schema_hint": str(schema_hint or "")[:160],
        "text_length": len(text or ""),
    }
    if error is not None:
        payload["error_type"] = error.__class__.__name__
        payload["error_message"] = str(error)[:240]
    logger.info("[evaluation-json-repair] %s", json.dumps(payload, ensure_ascii=False, sort_keys=True))
```

### Mudanças em `parse_json_with_repair`
1. Quando o parse falhar e ainda houver tentativas restantes:
- registrar `event="invalid_json_detected"`
- indicar `provider="azure_openai"` ou `provider="primary_ai"`

2. Após gerar o JSON corrigido:
- registrar `event="repair_generated"`

3. Quando esgotar tentativas:
- registrar `event="exhausted"`

### Benefícios
- visibilidade sobre taxa de JSON quebrado
- visibilidade sobre uso de reparo automático
- visibilidade sobre exaustão de tentativas
- tudo apenas no backend/logging interno
