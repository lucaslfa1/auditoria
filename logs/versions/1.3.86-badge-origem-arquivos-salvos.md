# v1.3.86 — Badge de origem (Auto/Manual) em Arquivos Salvos

**Data:** 2026-05-27
**Tipo:** UX — visibilidade de origem da auditoria
**Custo:** zero backend (campo já existia no payload)

## Contexto

Equipe de auditoria pediu (2026-05-27) badge visual indicando se cada item em Arquivos Salvos veio do ciclo automatizado ou foi upload manual — mesmo padrão que já existe em `Automação de Processos > Auditorias do Mês`. A motivação é controle: a auditoria quer revisar tudo (manual e automática) no mesmo fluxo, mas saber de relance qual é qual.

Decisão de não separar fluxo (auto vs manual em telas distintas) registrada em `memory/project_automacao_fluxo_unificado_ok.md`.

## Mudanças

### `src/shared/lib/auditOrigin.tsx` (novo)
- Util compartilhado com `AUTOMATION_ORIGINS`, `classifyOrigin()`, `isAutoOrigin()` e componente `OriginBadge`.
- Reproduz exatamente o padrão visual de `AuditoriasDoMes.tsx` (sky/Bot pra auto, violeta/User pra manual, slate pra desconhecida).
- Props: `criadoPor`, `size` (`sm`|`md`), `hideUnknown`, `hideOnMobile`.

### `src/features/saved-files/components/SavedFiles.tsx`
- Importa `OriginBadge`.
- Adiciona badge `size="sm" hideUnknown` na linha de meta-info do card, condicional a `item.audit_id` (uploads avulsos sem auditoria associada não exibem).
- Posicionado logo após o badge "Enviado" do `pending_approval`, mantendo a hierarquia visual.

## Decisões

- **Não refatorar `AuditoriasDoMes.tsx`** para usar o util agora — funciona como está, evitar risco de regressão visual. Refatoração fica como opcional.
- **`hideUnknown=true` em SavedFiles**: itens manuais existentes podem ter `criado_por=""` por backfill; mostrar "Origem desc." poluiria a UI sem agregar. Em `AuditoriasDoMes` o "Origem desc." é útil porque há filtro dedicado.
- **`size="sm"`**: card de SavedFiles é mais denso que o card de `AuditoriasDoMes`, escala `9px` casa com o badge "Enviado" pré-existente.
- **Condicional a `audit_id`**: arquivos soltos (uploads de texto/PDF sem ser auditoria) não têm origem auditável — esconde badge nesses casos.

## Como detectar "auto"

Critério (`AUTOMATION_ORIGINS` em `auditOrigin.tsx`):
```
{ automacao, cron, automation_engine, cloud_scheduler, resident_loop }
```
Validação: `core/automation.py:997` grava `criado_por="automacao"` ao persistir auditoria via ciclo. Telefonia manual usa o username do admin. Qualquer outro valor não-vazio cai em "manual".
