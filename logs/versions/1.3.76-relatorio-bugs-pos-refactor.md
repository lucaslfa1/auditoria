## TL;DR

O refactor MIT moveu modulos de `backend/` para subpastas (`core/`, `db/`, `utils/`, `audio/`, `storage/`, `jobs/`), mas **caminhos relativos a `__file__` nao foram corrigidos**. Resultado: **4 paths de producao apontam para diretorios inexistentes**, causando os sintomas reportados:

- Auditoria manual quebrada
- Triagem quebrada
- Qualidade da auditoria IA degradada
- Filtro anti-alucinacao do Whisper desativado silenciosamente

Ha tambem **import pollution** em 138 arquivos (`from repositories import audits` injetado sem uso).

Nenhum bug e arquitetural. Sao fixes pontuais e mecanicos.

## CRITICOS (BUG-001 a BUG-004) — afetam producao

### BUG-001: Triagem grava/le audios no path errado
- **Arquivo:** `backend/core/automation.py:72`
- **Codigo bugado:** `Path(__file__).resolve().parent / "storage" / "classified_audio"`
- **Resolve para:** `backend/core/storage/classified_audio/` (NAO EXISTE)
- **Deveria ser:** `backend/storage/classified_audio/` (existe, contem `.wav` reais)
- **Causa:** `automation.py` foi movido de `backend/` para `backend/core/`, mas calculo nao compensou o nivel adicional.
- **Impacto:** classificacao grava/busca arquivos no diretorio errado. Triagem aparenta "perder" audios.
- **Fix:** trocar `parent` por `parents[1]`

### BUG-002: Auditoria manual salva audios em path duplicado
- **Arquivo:** `backend/storage/audit_storage.py:69`
- **Codigo bugado:** `Path(__file__).resolve().parent / "storage" / "audits" / "audio"`
- **Resolve para:** `backend/storage/storage/audits/audio/` (duplicacao `storage/storage`)
- **Deveria ser:** `backend/storage/audits/audio/`
- **Impacto:** upload manual salva em pasta orfa; playback na UI nao acha o audio.
- **Fix:** trocar `parent / "storage"` por `parent` direto (ja estamos em `storage/`)

### BUG-003: filter_hallucinations virou no-op silencioso
- **Arquivo:** `backend/utils/text_processing.py:26`
- **Codigo bugado:** `_CONFIG_DIR = Path(__file__).resolve().parent / "config"`
- **Resolve para:** `backend/utils/config/` (NAO EXISTE)
- **Deveria ser:** `backend/config/`
- **Consequencia:** `_load_text_corrections()` retorna `{}` -> `_COMPILED_BLACKLIST=[]` e `_COMPILED_REPLACEMENTS=[]` -> filtro nao remove nada.
- **Reproducao:**
  ```
  TEXT_CORRECTIONS_CONFIG keys: []
  BLACKLIST patterns: 0
  filter_hallucinations('antropologo aqui') -> 'antropologo aqui'  (deveria remover)
  ```
- **Impacto:** alucinacoes do Whisper passam para a auditoria. Termos como "antropologo" e frases tipicas de URA contaminam transcricoes.
- **Fix:** `_CONFIG_DIR = Path(__file__).resolve().parents[1] / "config"`

### BUG-004: Golden dataset nao chega ao prompt da IA
- **Arquivo:** `backend/core/audit_evaluator.py:550`
- **Codigo bugado:** `os.path.join(os.path.dirname(__file__), "data", "rag_training", "exemplos_gabarito")`
- **Resolve para:** `backend/core/data/rag_training/...` (NAO EXISTE)
- **Deveria ser:** `backend/data/rag_training/exemplos_gabarito/` (existe)
- **Impacto:** `_get_golden_dataset_prompt_block()` retorna `""` sempre -> IA de auditoria avalia SEM os exemplos-gabarito -> qualidade despenca.
- **Fix:** subir 1 nivel: `os.path.join(os.path.dirname(os.path.dirname(__file__)), "data", ...)`

## GRAVES (BUG-005 a BUG-007) — estruturais

### BUG-005: Import pollution massivo
- **138 arquivos** Python contem `from repositories import audits` injetado no topo, frequentemente **sem usar** o simbolo.
- Exemplos: `core/network_utils.py`, `core/email_utils.py`, `core/config.py`, `core/export.py`, `core/export_fechamento.py`, `core/ai_feedback.py`, `tests/test_audit_evaluator_payloads.py`.
- Em `core/automation_rules.py` o import aparece **antes do docstring** (quebra convencao).
- **Causa:** algum script de refactor anterior ao `refactor_imports.py` injetou esse import como "polyfill".
- **Risco:** import circular latente, custo de inicializacao, leituras erradas em refactors futuros.
- **Fix:** script automatico inverso — remove `from repositories import audits` quando `audits` nao e referenciado no restante do arquivo.

### BUG-006: network_utils.py duplicado com propositos diferentes
- `backend/core/network_utils.py` (35 linhas) — contem `apply_dns_overrides()` (DNS patch).
- `backend/utils/network_utils.py` (45 linhas) — contem `should_trust_env_proxies()`, `create_requests_session()` (proxy/HTTP).
- **Mesmo nome, conteudo diferente.** `main.py:49` importa `core.network_utils` (correto); mas o mapping de `refactor_imports.py` aponta `network_utils` -> `utils.network_utils` — qualquer caller legado que use `from network_utils import ...` e redirecionado para o modulo ERRADO.
- **Fix recomendado:** renomear `utils/network_utils.py` para `utils/http_session.py` (ou `proxy_utils.py`), atualizar imports.

### BUG-007: __init__.py faltando em 5 pacotes
| Pacote | __init__.py |
|---|---|
| `backend/utils/` | FALTA |
| `backend/audio/` | FALTA |
| `backend/jobs/` | FALTA |
| `backend/storage/` | FALTA |
| `backend/scripts/` | FALTA |

Funcionam como namespace packages (PEP 420), mas quebram `patch.object()`, distribuicao em wheel, e o autodiscovery de algumas ferramentas. **Fix:** criar 5 arquivos vazios.
