# v1.3.141 — Extrai o dataset de benchmark de classification_review.py

**Data:** 2026-06-12
**Tipo:** Refatoração conservadora (sem mudança de comportamento)
**Custo:** nenhum
**Branch:** `main`

Continuação da refatoração para legibilidade (handover). `classification_review.py`
era o 8º maior arquivo do backend (~1535 linhas) misturando duas
responsabilidades distintas: a **fila de triagem** (`fila_revisao_classificacao`)
e o **dataset de benchmark** (gabarito humano). O próprio docstring já sinalizava
isso. Esta versão separa o benchmark.

## Método (ultracode)

Workflow de análise com 3 leitores paralelos antes de tocar no código:
estrutura+grafo de chamadas, superfície de fachada/monkeypatch, e cobertura de
testes. Conclusões que guiaram um corte seguro:
- Sem import circular (o módulo só importa `db.domain_constants` e
  `repositories.common`).
- **Nenhuma função pública chama outra pública** → mover não quebra monkeypatch.
- Os testes patcham `db.database.X` / `main.database.X` (fachada), nunca
  `repositories.classification_review.X` para afetar caller → basta manter os
  nomes reexportados.
- ⚠️ A análise automática subestimou um uso: `extract_returning_id` também é
  usado por `sincronizar_fila_revisao_classificacao` (fila, fica). Confirmado por
  grep manual antes de remover imports — `extract_returning_id` foi mantido.

## O que mudou

- **NOVO** `backend/repositories/classification_review_benchmark.py`: 6 funções do
  dataset de ground truth (`upsert_ligacao_auditada`,
  `get_ligacao_auditada_por_hash`, `registrar_resultado_classificacao`,
  `registrar_resultado_auditoria`, `get_resumo_ligacoes_auditadas`,
  `listar_ligacoes_auditadas`). Elas estavam espalhadas no topo e no fim do
  arquivo — agora num módulo coeso.
- `_normalize_sector_id` → movido para `repositories/common.py` como
  `normalize_sector_id` (junto da família `normalize_*`), pois é compartilhado
  entre fila e benchmark. Importado com alias `_normalize_sector_id` nos dois
  módulos para manter os call sites byte-idênticos.
- `classification_review.py` reexporta as 6 funções (compat total:
  `repositories.classification_review.X` e a fachada `db.database` seguem
  válidos). Removidos os imports que só o benchmark usava
  (`normalize_quality_reference`).
