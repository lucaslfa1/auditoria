# v1.3.137 — Limpeza de classification.py: equivalência de alertas extraída + dead code

**Data:** 2026-06-12
**Tipo:** Refatoração conservadora (sem mudança de comportamento) + remoção de dead code
**Custo:** nenhum
**Branch:** `main`

Continuação da refatoração para legibilidade (dev frio). `classification.py` era
o maior arquivo do backend (1970 linhas). Esta versão tira ~230 linhas dele.

## O que mudou

### 1. Dead code removido
`_legacy_align_classification_with_catalog_unused` (~107 linhas) — versão antiga
do alinhamento com o catálogo, substituída por `align_classification_with_catalog`.
Tinha **zero referências** (confirmado por grep em `backend/` e `tests/`) e ainda
um **bug latente**: referenciava `first_alert` antes da definição (NameError se
algum dia fosse chamada). Some o "qual função de align é a real?" para quem chega.

### 2. Equivalência de alertas extraída para módulo próprio
Novo `backend/core/classification_alert_equivalence.py` com o conceito coeso
"mesmo tipo de ocorrência, prefixo de setor diferente" (ex.: `UTI-POSICAO-MOT`
↔ `DISTRIBUICAO-POSICAO-MOT`):
- Constantes `_OPERATIONAL_SECTORS`, `_OPERATIONAL_ALERT_PREFIXES`,
  `_LOGISTICA_KIND_ALERTS`.
- Funções `_get_catalog_alert`, `_apply_alert_match`, `_normalize_alert_token`,
  `_infer_alert_actor_suffix`, `_extract_operational_alert_kind`,
  `_operational_alert_suffix`, `_get_equivalent_alert_for_sector`.

Mesma estratégia da v1.3.133 (`classification_audio`/`classification_lexicon`):
os nomes são **reexportados** de `core.classification`, então qualquer caller
externo e os testes que acessam `classification._OPERATIONAL_SECTORS` /
`classification._OPERATIONAL_ALERT_PREFIXES` (test_bbm_distribuicao_migration)
continuam funcionando sem alteração.

### Preservação do monkeypatch
`_get_equivalent_alert_from_context` **fica** em `classification.py` porque os
testes (`test_strict_rh_sector_enforcement`) o patcham diretamente.
`_get_catalog_alert`/`_apply_alert_match` resolvem `load_audit_criteria_catalog`
em RUNTIME (`from core import classification`) para honrar o patch dessa função
nos testes de align (`test_audit_pipeline_context`, `test_review_queue_contract`)
e evitar import circular no carregamento.
