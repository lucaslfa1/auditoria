**Data:** 2026-06-16
**Tipo:** Refatoração conservadora (legibilidade p/ handover)
**Custo:** nenhum
**Branch:** `main`

## O que foi feito

Extraído de `backend/core/audit_evaluator.py` (806 linhas) o cluster dominante de
**construção do prompt de avaliação** para o módulo irmão
`backend/core/audit_evaluation_prompt.py` (~360 linhas movidas, ~45% do arquivo).

Símbolos movidos (10):
- `AUDIT_EVALUATION_SCHEMA_HINT` (hint de formato do prompt)
- `get_audit_system_prompt` (orquestra a montagem) + os blocos:
  `_build_criteria_text`, `_build_audit_evaluation_response_format`,
  `_build_strict_evidence_retry_prompt`, `_build_default_evaluation_user_prompt`,
  `_ensure_evidence_contract_in_user_prompt`, `_should_apply_password_rule`,
  `_build_diarization_prompt_block`, `_get_golden_dataset_prompt_block`

O restante de `audit_evaluator.py` (orquestração `evaluate_transcription`/
`evaluate_with_azure`/`evaluate_with_ai_priority`, `AuditEvaluationDependencies`,
knobs de env, `_build_feedback_query_embedding`) consome o cluster por chamada de
função — segue funcionando via reexport.

## Compatibilidade

Os 10 nomes são reexportados de `core.audit_evaluator`
(`from core.audit_evaluation_prompt import (...)  # noqa: E402,F401`). `evaluate_*`,
a fachada `core/evaluation.py`/`services.py` e os testes
(`test_audit_prompt_sector_rules`, `test_procedimentos_rag`,
`test_audit_evaluation_wrappers`) continuam usando `audit_evaluator.<nome>`.

## Prova

- `ast.dump` byte-a-byte das 9 funções + igualdade da constante: idêntico
  (`HEAD:audit_evaluator.py` × novo módulo).
- Identidade de objeto no reexport para os 10 símbolos.
- Sem ciclo: o novo módulo importa só `procedimentos_rag`/`audit_rules`/`schemas`
  (nenhum importa `audit_evaluator`); o type hint `AuditEvaluationDependencies`
  fica em `audit_evaluator` e é importado só sob `TYPE_CHECKING` (com
  `from __future__ import annotations`, não é avaliado em runtime). `glob` é import
  local (move junto com `_get_golden_dataset_prompt_block`).
- Análise de nomes-livres validou o conjunto de imports do novo módulo.
- Suite backend completa (banco de teste; nunca prod): verde.
