# Versão 1.3.73 — Fixes Críticos da Revisão Técnica

**Data:** 18 de Maio de 2026
**Autores:** Lucas Afonso

## Contexto
Revisão técnica multi-agente dos módulos de transcrição, classificação e organização dos AI modules no Sidebar identificou **8 itens críticos**. Esta versão resolve todos os 8.

## Itens resolvidos

### Transcrição (`backend/core/transcription.py`, `backend/transcription_providers/*`)

**#1 — Eliminada dupla conversão WAV em `hybrid_dual`**
- `transcription_providers/azure.py` e `openai_diarize.py` só convertem para WAV se o `mime_type` recebido não for `audio/wav`/`audio/x-wav`/`audio/wave`.
- Em `hybrid_dual`, o áudio já vem normalizado por `prepare_audio_for_azure`; agora os providers respeitam isso, evitando 2× conversão paralela.

**#2 — `hybrid_dual` sobrevive a falha parcial**
- `asyncio.gather(..., return_exceptions=True)` no `run_hybrid_dual`.
- Se Diarize falhar: usa Whisper isolado e retorna `effective_strategy="whisper"`.
- Se Whisper falhar: usa Diarize isolado e retorna `effective_strategy="gpt4o_diarize"`.
- Se ambos falharem: levanta `RuntimeError` para o orchestrator tentar próxima estratégia.

**#3 — Bonus do `hybrid_dual` calibrado**
- Bonus reduzido de **50000 → 1500** (constante `HYBRID_DUAL_BONUS`), proporcional à escala real de score (~3000–15000).
- Removido override `if len(result) > 2: is_valid = True` — agora o gate em `_transcription_candidate_is_acceptable` se aplica uniformemente a todas as estratégias.
- Bonus só é aplicado quando a fusão real aconteceu (`effective_strategy == "hybrid_dual"`).

**#4 — Logs mais transparentes em falhas de Whisper/Fast**
- `transcription_providers/azure.py` adiciona `logger.warning` antes do `raise`, indicando explicitamente que o orchestrator tentará a próxima estratégia. Resolve a "falha silenciosa" reportada.

**#5 — Metadata fiel quando merge falha**
- `merge_transcriptions_with_gpt4o` agora retorna tupla `(segments, merge_status)`:
  - `"merged"` — fusão GPT-4o concluída.
  - `"no_credentials"` — Azure OpenAI não configurado.
  - `"merge_failed"` — exceção/parse falhou.
- `run_hybrid_dual` propaga o `effective_strategy` correto (`hybrid_dual` apenas quando fundido; caso contrário, `gpt4o_diarize`).
- `attempts[*]` ganha novo campo `effective_strategy` (além do `strategy` solicitado) para observabilidade.

### Classificação (`backend/classification.py`, `backend/automation.py`)

**#6 — Alias `BAS-POLICIAL` resolvido antes de `get_criteria` + fallback YAML**
- Nova função pública `classification.canonicalize_alert_id()` (wraps o `_canonicalize_alert_id` interno).
- `automation._build_alert_from_classification` resolve alias **antes** de `get_criteria`, persistindo o `id` canônico no `AuditAlert` retornado.
- Quando o DB retorna `[]` em runtime (não-pytest), faz fallback para `scoring_rules.yaml` via `db.scoring_loader.get_alerts()`. Se nem o YAML tiver critérios, `logger.error` claro.

**#7 — Guardrail `direction_mismatch` reativado (per-alert keyword)**
- Decisão A.1: regra inferida por keyword no `alert_id`, sem nova coluna no DB (fica como pendência 🟡).
- Nova função `_expected_direction_for_alert(alert_id)`:
  - `PRIORITARIO-POLICIA`/`PARADA-MOT`/`DESVIO`/`POSICAO-MOT`/`PONTO-APOIO` → `efetivada`.
  - `RECEPTIVA`/`RECEBIDA`/`INBOUND` → `receptiva`.
- Comportamento: marca `needs_review` com motivo `direction_mismatch`, **não bloqueia** a auditoria.
- Bypass de emergência: env `DIRECTION_GUARDRAIL_ENABLED=false`.
- Call-site em `enforce_operator_and_direction_guardrails` agora usa o `alert_id` final em vez do `sector_id`.

### Frontend (`src/shared/components/Sidebar.tsx`)

**#8 — View `admin` reconectada à UI (decisão B.1)**
- Acordeão "Inteligência Artificial" agora tem **4 sub-itens** na ordem:
  1. **Critérios** → `'admin'` (`AdminCriteriaPage` — CRUD completo com motivo/audit log)
  2. **Feedback IA** → `'ia'` (`AIFeedbackPage` — feedbacks/instruções para IA)
  3. **Apelidos de Setor** → `'admin-aliases'`
  4. **Prompts de IA** → `'admin-prompts'`
- Acessibilidade: adicionado `aria-expanded`, `aria-controls="ai-submenu"`, `role="group"` e `aria-label` no submenu.
- Cor de ativo padronizada (`text-primary-300`, alinhada com itens-pai).
- Lista `isActive` do botão pai inclui `'admin'`.
