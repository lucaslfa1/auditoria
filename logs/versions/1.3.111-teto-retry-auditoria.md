# v1.3.111 - Auditoria automática sem retry; falha → descarte permanente

**Data:** 2026-06-10
**Tipo:** Custo / Automação / Política
**Status:** implementado, testes-chave 12/12, commitado e enviado para `main` (deploy via Actions)

## Política (decisão do Lucas, 2026-06-10)

> "Auditar é só uma vez. 'Retriar' não existe."
> - **Fluxo automático:** se falhar, **não dá retry e é descartada (permanente)**.
> - **Fluxo manual:** se falhar, aparece a edição para **editar manualmente** (humano).

A automação **não re-audita** item que falhou. Uma passada só; falhou →
tombstone permanente. Cada re-transcrição = gasto Azure, então zerar o retry +
descartar de vez corta o reprocessamento.

## Mudança

**`backend/core/automation_disposition.py`** — `_transient_retry_limit()`:
default **3 → 1** (= **sem re-tentativa**; a 1ª falha já esgota).

**`backend/core/automation_cache.py`** — no gatekeeper de **erro de transcrição**
(`handle_transcription_runtime_error`), o descarte ao esgotar passou de
`DISCARD_RECOVERABLE` para **`DISCARD_IMPOSSIBLE`** (tombstone permanente), via a
flag já existente `AUTOMATION_DISCARD_IMPOSSIBLE_TRANSCRIPTION` (default ON) —
mesmo tratamento que a transcrição **vazia** já tinha. Com `limite=1`, a 1ª falha
de transcrição cai direto aqui.

**`backend/.env.example`** — `AUTOMATION_TRANSIENT_RETRY_LIMIT=1`.

## Efeito

- **Automático, falha de transcrição/avaliação** → **tombstone permanente** na 1ª
  falha. Não re-transcreve, não vai para triagem manual, não volta.
- **Automático, falha de infra** (timeout/áudio ausente, `_handle_transient_failure`)
  → **descarte recuperável** (mantido de propósito): é genuinamente transitório, e
  no D-1 não reaparece sozinho — permanente aqui só arriscaria perder ligação boa
  por blip de rede. Avisar o Lucas se quiser permanente também.

## ⚠️ Atenção para outros agentes

Para esta política, manter os defaults: `AUTOMATION_TRANSCRIPTION_FAILURE_RETRY=true`
(branch moderno que descarta ao esgotar) **+** `AUTOMATION_DISCARD_IMPOSSIBLE_TRANSCRIPTION=true`
(descarte permanente). O "sem retry" vem do **`AUTOMATION_TRANSIENT_RETRY_LIMIT=1`**,
NÃO de desligar a flag de retry (desligá-la roteia para `needs_manual_triage` legado).

## Fluxo manual

"Falha → editar manualmente" é o caminho do humano (auditoria manual), separado do
gatekeeper de automação. Comportamento existente; nada a mudar.
