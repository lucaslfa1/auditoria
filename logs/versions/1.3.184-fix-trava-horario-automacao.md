# v1.3.184 — Fix: "Horário de início" travava a esteira + relabel do campo

**Data:** 2026-06-18
**Tipo:** Correção de bug (config de produção) + UX
**Custo:** nenhum
**Branch:** `main`

## Sintoma

Agendamento configurado (meta 40, horário 20:50) **não executava**: a esteira de
automação Huawei não baixava/auditava nada havia dias.

## Causa raiz

O disparo real é um **Cloud Scheduler externo 1x/dia às 06:30 BRT** (09:30 UTC —
confirmado em `automation_cycle_runs`, todos `source=cloud_scheduler` às 09:30Z). O
campo de UI "Horário de início" (`huawei_d1_horario_execucao`) **não é o gatilho** —
é só um GATE interno em `_deve_executar` ([huawei_d_minus_1.py:614](../../backend/core/huawei_d_minus_1.py)):
na 1ª tentativa do D-1, `if now < horario_execucao: return False, "antes_do_horario_programado"`.

A Fátima setou o campo para **20:50**. Como o cron passa às 06:30 e `06:30 < 20:50`,
o pipeline pulava todo dia com `antes_do_horario_programado`, e como o cron só roda
1x/dia (não volta às 20:50), o lote **nunca rodava**. O label "Hora em que a automação
inicia" induziu ao erro: sugere gatilho, mas o campo só consegue **adiar** o disparo
fixo — nunca antecipá-lo.

## O que foi feito (item 1 de 3 do plano de correções)

1. **Dado (desbloqueio imediato, prod):** `huawei_d1_horario_execucao` `20:50` → `06:00`
   via `configuracoes` + trilha em `configuracoes_audit_log` (`origem=system`,
   `alterado_por=system:claude-fix`). `valor_antes` confirmado `20:50` antes de gravar.
   `06:00 ≤ 06:30` (cron atual) **e** `≤ 20:00` (cron-alvo), então é seguro nos dois
   estados: a esteira volta a rodar já no próximo cron e não trava quando o horário do
   scheduler mudar.
2. **UI (este commit):** campo renomeado de "Horário de início" → **"Não iniciar antes
   de"**, hint de "Hora em que a automação inicia" → "Trava de segurança — o disparo
   diário é agendado na infraestrutura, não por este campo"
   ([AutomationConfigPanel.tsx](../../src/features/automacao/components/AutomationConfigPanel.tsx)).
   Comunica que o campo é GATE, não gatilho — fecha a armadilha que derrubou a esteira.

## Handoff (infra — Gemini/GCP, fora deste commit)

Para o disparo passar a ocorrer **às 20:00** (em vez de 06:30), mover o cron do(s)
job(s) do Cloud Scheduler (1x/dia preservado — custo seguro):

```bash
gcloud scheduler jobs describe auditoria-automacao-run --location=southamerica-east1
gcloud scheduler jobs update http auditoria-automacao-run \
  --location=southamerica-east1 --schedule="0 20 * * *" --time-zone="America/Sao_Paulo"
```
Conferir se há um 2º job apontando p/ `/api/telefonia/cron/sync` e alinhar (coletor
antes do auditor). O time-zone evita erro de UTC.
