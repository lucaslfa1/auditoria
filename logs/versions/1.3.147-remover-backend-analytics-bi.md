# v1.3.147 — Remove o backend de analytics/BI (router + PowerBI)

**Data:** 2026-06-12
**Tipo:** Remoção de feature (decisão de produto)
**Custo:** nenhum
**Branch:** `main`

Continuação da remoção do módulo BI (após o frontend na v1.3.146). Decisão:
remover o router de analytics inteiro, incluindo o export PowerBI.

## Removido

- `backend/routers/analytics.py` (router `/api/analytics/*`: indicators,
  trend, top-failures, matching-quality + `/external/powerbi/*`). Estava
  registrado só no `main.py`; sem consumidor frontend após a v1.3.146.
- `backend/repositories/report_powerbi.py` (exclusivo do router).
- `backend/repositories/matching_report.py` (exclusivo do router).
- `main.py`: import + `include_router(analytics_router)`.
- `tests/backend/test_e2e_pipeline.py`: smoke-import de `routers.analytics`.

## Mantido (de propósito — NÃO era órfão)

- **`backend/repositories/analytics_quality.py`**: apesar do nome "analytics",
  é usado pelo **Portal do Supervisor** (`supervisor.py` →
  `get_indicators_by_supervisor`). NÃO faz parte do BI removido; fica.
- **`@router.get("/api/analytics")` em `routers/system.py`** (`get_audit_analytics`
  → `database.get_analytics`): endpoint do router core (system), separado do
  router analytics. Ficou sem consumidor frontend (o Dashboard usava), mas é do
  router core e não foi tocado nesta remoção. Pode ser removido depois se desejado.

## Validação

- `main.py` importa OK sem o analytics_router.
- Suite backend completa verde (banco de teste local; nunca prod).
- `repositories.analytics_quality` segue importável (Supervisão intacta).

## Resumo da remoção do módulo BI (v1.3.146 + v1.3.147)

- Frontend: telas Dashboard de Qualidade + PerformanceDashboard removidas.
- Backend: router `/api/analytics` + PowerBI export + 2 repos exclusivos removidos.
- Intocado: Fechamento e seu export p/ BI externo; `/api/dashboard/*` (store de
  auditorias salvas); `analytics_quality` (Supervisão).
