# v1.3.166 — Extrai export do catálogo de critérios de admin_criteria.py

**Data:** 2026-06-16
**Tipo:** Refatoração conservadora (legibilidade p/ handover)
**Custo:** nenhum
**Branch:** `main`

## O que foi feito

Extraído de `backend/repositories/admin_criteria.py` (915 linhas) o cluster de
**export/serialização do catálogo de critérios** (formato `auditCriteria.json`)
para o módulo irmão `backend/repositories/admin_criteria_export.py` (~145 linhas).

Símbolos movidos (4):
- `get_export_format` (monta a hierarquia setor→alerta→critério + regra POP 4.1
  dos setores de risco que compartilham alertas com `bas`)
- `_safe_select_rows`, `_get_existing_columns` (SELECT defensivo via
  `psycopg2.sql`, tolerante a colunas ausentes em DBs antigos)
- `_with_row_factory` (movido junto e **reexportado de volta** — opção que quebra
  o ciclo de import, já que os getters/CRUD que ficam em `admin_criteria` o usam)

## Compatibilidade

Os 4 nomes são reexportados de `repositories.admin_criteria`
(`from repositories.admin_criteria_export import (...)  # noqa: E402,F401`). O
`routers/admin_criteria.py` (importa `get_export_format`), os getters
(`get_sectors/get_alerts/get_criteria/get_sector_members`) e o CRUD continuam
usando `admin_criteria.<nome>`.

## Prova

- `ast.dump` byte-a-byte das 4 funções: idêntico (`HEAD:admin_criteria.py` × novo
  módulo).
- Identidade de objeto no reexport: `admin_criteria.<nome> is admin_criteria_export.<nome>`.
- Sem ciclo de import (`admin_criteria_export` não importa `admin_criteria`).
- Suite backend completa (banco de teste; nunca prod): verde.

## Contexto

P3 do plano de extração conservadora. `admin_criteria.py` já havia perdido
`admin_criteria_audit_log` (v1.3.157). `get_export_format` tem teste de
comportamento dedicado (`test_admin_criteria_repository.py`). Nenhum símbolo do
cluster é monkeypatchado (os únicos patches em `admin_criteria.*` são
`get_criteria`/`get_alerts`, que ficam no módulo de origem).
