## Objetivo

Implementar a orquestração do Caminho 1 (A Via Expressa) descrita no plano de Automação Híbrida. O objetivo foi criar o motor que roda em background no backend e a interface visual para o perfil Admin controlar o ligamento/desligamento do motor, bem como visualizar métricas de ligações baixadas e auditorias geradas de forma autônoma.

## Mudancas

### Backend

- `backend/automation_engine.py` (novo):
  - Motor de automação `asyncio` rodando em background.
  - O loop chama sequencialmente a coleta da Huawei (`executar_sync_huawei`) e o envio para a IA (`audit_all_pending`).
  - O estado LIGADO/DESLIGADO é salvo no banco (`configuracoes`).
  - Pausas e falhas são controladas nativamente pelo loop sem bloquear o servidor.
- `backend/main.py`:
  - Modificado o evento de `lifespan` do FastAPI para inicializar o motor de automação (`start_engine()`) ao subir o servidor e parar adequadamente no encerramento.
- `backend/routers/automation.py`:
  - Inclusão dos endpoints `/engine/toggle` (POST) para controlar o estado do motor e `/engine/status` (GET) para leitura do progresso, baixadas e auditadas pela tela.

- `src/shared/components/Sidebar.tsx` e `src/App.tsx`:

## Como validar

1. Acesse o sistema localmente (ou pós-deploy) com o perfil `admin`.
2. No menu lateral, clique em **Automação**.
3. Verifique o status ocioso.
4. Clique no controle LIGAR. O motor irá rodar a primeira sincronização de telefonia (dependendo do .env, caso ENABLE_HUAWEI_SYNC=true baixará reais) e passará tudo pela IA, jogando os itens para a aba de rascunhos.
5. Verifique a listagem no frontend acompanhando os números na aba Automação.
