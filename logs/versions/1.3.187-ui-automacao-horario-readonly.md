# v1.3.187 — UI Automação: campo de horário read-only + "Como usar" explica o ciclo

**Data:** 2026-06-18
**Tipo:** UX (ajuste pós-feedback do Lucas)
**Custo:** nenhum
**Branch:** `main`

## Contexto

Após a v1.3.184 (campo virou "Não iniciar antes de"), o Lucas pediu para o campo
deixar de ser editável e apenas mostrar quando o ciclo roda, e para a caixa
"Como usar" explicar o horário das automações.

## O que foi feito (frontend only)

1. **Campo de horário** ([AutomationConfigPanel.tsx](../../src/features/automacao/components/AutomationConfigPanel.tsx)):
   - label "Não iniciar antes de" → **"Automações"**;
   - hint → **"Horário em que o ciclo de automações começa"**;
   - input **read-only / disabled** (sem `onChange`/`onBlur`) — o disparo é agendado
     na infraestrutura (Cloud Scheduler), não editável pela tela. O valor reflete o
     horário do agendamento (`huawei_d1_horario_execucao`).
2. **"Como usar"** ([AutomacaoPage.tsx](../../src/features/automacao/components/AutomacaoPage.tsx)):
   - passo 1 "Defina a meta de auditorias e os horários do ciclo." → "Defina a meta
     de auditorias do ciclo." (horário não é mais setado aqui);
   - passo 3 agora explica: "O ciclo roda automaticamente 1x por dia, no horário
     mostrado em Configurações; acompanhe o progresso…".

## Importante — sincronia display↔cron

O campo mostra `huawei_d1_horario_execucao` (hoje **06:00**, cron real ~06:30). Como
o campo é só leitura e reflete o agendamento, ao mover o cron para 20:00 (handoff
Gemini, v1.3.184) é preciso setar **`huawei_d1_horario_execucao=20:00` junto**, senão
o campo mostra um horário e o cron dispara em outro. É a contrapartida da abordagem
"Fixar" (ver [[project-automacao-agendamento-fixar]] na memória).
