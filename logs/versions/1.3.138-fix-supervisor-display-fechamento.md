# v1.3.138 — Fix de exibição do supervisor no fechamento (follow-up de revisão de código)

**Data:** 2026-06-12
**Tipo:** Fix de UI (display-only)
**Custo:** nenhum
**Branch:** `main`

## Contexto

Na revisão do commit `74a2f10e` ("usa supervisores cadastrados"), o
campo Supervisor do fechamento virou read-only (`disabled`), com a fonte de
verdade no cadastro do colaborador — decisão correta (supervisor é relação
estrutural, não override mensal). Mantida.

## Bug encontrado e corrigido

O commit removeu a `<option>` de fallback do select. Como o `GET
/api/fechamento/supervisores` lista apenas supervisores **ativos**, uma linha
cujo supervisor cadastrado não esteja nessa lista (ex.: supervisor inativo mas
ainda vinculado) exibia o campo **em branco** na tela, mesmo com o valor
correto vindo do backend (o Excel sempre saiu certo — é só display).

**Fix:** re-adiciona a `<option>` de fallback com o valor de `row.supervisor`
quando ele não está em `supervisorOptions`. Mantém o select desabilitado.
