# v1.3.133 — Divisão dos 3 maiores arquivos do backend

**Data:** 2026-06-12
**Tipo:** Refatoração estrutural (movimentação pura, zero mudança de comportamento)
**Custo:** nenhum em runtime
**Branch:** `main`

## Motivação

Pergunta do Lucas: "um time que nunca viu uma linha desse código" navega bem
em arquivos de 2k+ linhas? Resposta honesta: não é o ideal — e o cálculo de
risco que justificou NÃO dividir (v1.3.127) mudou: a suíte está verde (836)
e temos provas objetivas (snapshot de rotas, fidelidade byte-a-byte).

## 1. `routers/telefonia.py`: 2.106 → ~1.199 linhas

- Orquestrador mantém: estado de módulo (`_LAST_SYNC*`, `_ACTIVE_AUDIT_TASKS`),
  todos os helpers, `router` com prefix, e a montagem (include_router no fim).
- 27 endpoints movidos para `routers/telefonia_routes/`:
  `sync.py` (11), `cron_d1.py` (5), `recordings.py` (6 incl. debug OBS),
  `audit_actions.py` (5 + `AuditRecordingRequest`).
- Subrouters referenciam helpers/estado via `tf.<nome>`
  (`from routers import telefonia as tf`) — resolução em runtime preserva
  monkeypatch dos testes e o estado compartilhado. `global X` dos endpoints
  movidos reescrito como `tf.X = ...`.
- Endpoints e modelo reexportados no orquestrador (testes chamam
  `telefonia.cancel_sync(...)` direto; `routers/automation.py` importa
  `sync_manual`).
- **Prova:** snapshot ordenado das 27 rotas `(path, métodos)` idêntico
  antes/depois; compilação limpa.

## 2. `db/database.py`: ~1.955 → 1.749 linhas

- `db/saved_audits.py` (374): todo o bloco do espelho de Arquivos Salvos
  (builders `_build_saved_audit_*` + `_sync_arquivo_salvo_for_audit_inline`).
- `db/audit_media.py` (320): anexo/recuperação de áudio +
  `persist_audit_artifacts` (funil único de persistência).
- Chamadas de volta à fachada via `from db import database as dbm` DENTRO das
  funções — evita ciclo de import e preserva os patches dos testes em
  `db.database.*` (verificado com smoke de monkeypatch real).
- Reexports na fachada para TODOS os nomes movidos (públicos e privados).
- **Prova:** fidelidade byte-a-byte dos corpos movidos vs HEAD (revertendo só
  as substituições `dbm.` permitidas); 8 atributos críticos presentes.
- 🐛 Registro de bug latente PRÉ-EXISTENTE encontrado na extração:
  `get_audit_by_id` era chamado como nome livre (inexistente) em 2 branches
  mortos (`recover_audit_audio_from_classified_queue` com `audit` sempre
  passado; `finalize_contestation_review` que é código morto — o router usa
  o repository direto). Mantido o comportamento (crash em ambos os casos);
  candidato a limpeza futura.

## 3. `core/classification.py`: −195 linhas

- `core/classification_lexicon.py` (162): as 15 constantes do "léxico de
  sinais" (keywords, pesos, contextos) — dados puros.
- `core/classification_audio.py` (74): `truncate_audio`,
  `_trim_leading_silence_enabled`, `MAX_AUDIO_DURATION_SECONDS`,
  `get_mime_type`.
- Reimport explícito nome a nome em classification.py — todos os caminhos
  `from core.classification import X` continuam válidos (automation, huawei,
  telefonia, testes).
- Funções patchadas por testes NÃO foram movidas
  (`align_classification_with_catalog`, `resolve_operator_identity`,
  `load_audit_criteria_catalog`, `_resolve_db_sector_alias`,
  `_get_equivalent_alert_from_context`).
