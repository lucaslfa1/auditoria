# v1.3.153 — Extrai o parser de JSON com reparo de evaluation.py

**Data:** 2026-06-15
**Tipo:** Refatoração conservadora (sem mudança de comportamento)
**Custo:** nenhum
**Branch:** `main`
**Fase:** Plano de arquitetura — corte P1 (1/4)

`core/evaluation.py` misturava o subsistema de **reparo de JSON** com a lógica de
avaliação. Esse subsistema já era importado também por `core/transcription.py` e
`core/summary_regeneration.py` — extraí-lo deduplica uma dependência real.

## O que mudou

- **NOVO** `core/json_repair.py`: cluster de 5 funções (256-455 do antigo
  `evaluation.py`): `_log_json_repair_event`, `_iter_local_json_candidates`,
  `_try_parse_json_locally`, `_build_azure_json_repair_client`,
  `parse_json_with_repair`. Imports próprios (`json`, `logging`, `re`, `typing`,
  e as constantes de `core.config`); imports lazy (`openai`, `core.config.GENERATION_CONFIG`,
  `core.cost_guard`) preservados dentro das funções.
- `core/evaluation.py` **reexporta** `parse_json_with_repair` (mantém
  `from core.evaluation import parse_json_with_repair` funcionando para
  `transcription`, `summary_regeneration`, `test_json_repair` e o uso interno em
  `_get_audit_evaluation_dependencies`). `parse_json_with_repair` segue em `__all__`.
  Removida a constante `AI_MODEL` do import de `core.config` (só era usada pelo
  cluster movido).

## Por que era seguro

- **Cluster contíguo e fechado**: as 5 funções só chamam umas às outras + lêem
  constantes de `core.config`. Nenhuma chama função que fica em `evaluation.py`.
- **Sem monkeypatch de namespace**: nenhum teste faz patch em
  `core.evaluation.parse_json_with_repair`; os testes injetam um parser fake via
  dependency injection (`parse_json_with_repair=lambda...`) e `test_json_repair.py`
  importa e chama a função real (coberto pelo reexport).
- **Sem ciclo**: `core.config` não importa `evaluation`/`json_repair`.
- Apenas `parse_json_with_repair` é público (em `__all__`); os 4 helpers
  `_` não têm importador externo — movidos por inteiro.

## Prova de equivalência

- `ast.dump` das 5 funções: HEAD (`evaluation.py`) == `json_repair.py` — **5/5
  idênticas**.
- Reexport por identidade: `evaluation.parse_json_with_repair is json_repair.parse_json_with_repair`;
  idem para `transcription.parse_json_with_repair`.
- Import smoke OK; `AI_MODEL` removido do namespace de `evaluation`.

## Validação

- Suite backend completa (banco de teste local; nunca prod): verde.

## Resultado

`evaluation.py`: 793 → 596 linhas; subsistema de reparo de JSON isolado e
deduplicado. Próximos cortes P1: `audit_drafts`, `colaboradores_audit_log`,
