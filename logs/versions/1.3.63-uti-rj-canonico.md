## Objetivo
- Aglutinar tambem as sub-escalas `RJ - *` (UTI de RJ) no setor canonico `uti` no fluxo de classificacao/auditoria.

## Contexto
A v1.3.61 corrigiu BAS/UTI aglutinando sub-escalas via `startswith("bas")` / `startswith("uti")`. Faltou contemplar os operadores cadastrados com `setor = "RJ - AZUL"`, `"RJ - Amarela"`, `"RJ - CINZA"`, `"RJ - VERDE"` (todos com `organizacao_telefonia = "UTI"`, ou seja, UTI de RJ).

Como o setor cru nao comeca com `"uti"` nem `"bas"`, `map_db_sector_to_classification_sector` devolvia `None`, e `_get_effective_db_sector` caia no fallback retornando a string crua `"RJ - AZUL"` — que aparecia na UI ao baixar/classificar ligacoes da telefonia e ficava orfa do catalogo de criterios.

## Sintoma
Apos baixar ligacoes da Huawei e classificar, o campo de setor exibia `"RJ - AZUL"` (ou outra cor) em vez de `UTI`.

## Causa Raiz
`backend/repositories/operators.py`:
- Linha 233: `if normalized_setor.startswith("uti"):` nao cobria prefixo `rj`.
- Linha 277: idem para `_matches_operador_sector`.

`backend/data/operadores_seed.json` armazena (verificado):
- `RJ - AZUL`
- `RJ - Amarela`
- `RJ - CINZA`
- `RJ - VERDE`

Nenhum outro setor canonico comeca com `rj` (apenas UTI usa esse prefixo no seed).

## Mudancas

### Backend
- **`backend/repositories/operators.py:233-236`** em `map_db_sector_to_classification_sector`:
  ```python
  if normalized_setor.startswith("uti") or normalized_setor.startswith("rj"):
      return "uti"
  ```
- **`backend/repositories/operators.py:277-283`** em `_matches_operador_sector`: mesma extensao para manter simetria.

## Principio Adotado (reforco do v1.3.61)
- **Sistema (classificacao + auditoria)** → usa apenas id canonico (`uti`, `bas`). Cor/RJ nao discrimina criterios.
- **Fechamento** → preserva granularidade. `UTI (RJ)` e tratado a parte pela tabela de pontuacao editavel manualmente (ja OK, nao alterado).

## Impacto
- Ligacoes de operadores `RJ - *` passam a aparecer com setor `UTI` na classificacao.
- Auditorias casam com criterios do setor `uti`.
- Fechamento **inalterado**.

## Riscos
- Baixo: `startswith("rj")` so colide com UTI no seed atual (verificado via grep).
