# v1.3.83 — Fix 500 em /api/audit/{id}/audio (TypeError no kwarg `file_hash`)

**Data:** 2026-05-27
**Tipo:** Bugfix urgente
**Custo:** zero (rollback de mudança meio-aplicada)
**Bug relacionado:** BUG-033 (toast "Erro no Servidor / Request failed (500)" ao abrir áudio em Arquivos Salvos)

## Contexto

Usuário (Lucas) reportou: ao abrir uma auditoria em "Arquivos Salvos" dentro de "Automação de Processos", o painel renderiza mas dispara toast global "Erro no Servidor". DevTools mostrou `GET /api/audit/3/audio` retornando HTTP 500.

A auditoria #3 tem áudio gravado no banco:
```
audio_storage_path:        2026/05/audit_3_bb536d3a69bd.wav
audio_original_filename:   PARADA-MOTORISTA-agent-11406-1_10_2025_8_19_35-node01-1759317573-76676.wav
audio_size_bytes:          1.526.764
audio_mime_type:           audio/wav
```

Ou seja: **o áudio existe**, o problema é puramente o endpoint quebrar antes de chegar a lê-lo.

## Causa raiz

Commit `69f9e3a1` (2026-05-27 08:49, co-autorado com Gemini CLI) alterou `_resolve_existing_audit_audio_path` em `backend/routers/audit.py:45` para passar um kwarg `file_hash` ao chamar `resolve_stored_audit_audio_path`:

```python
audio_path = resolve_stored_audit_audio_path(
    media_record.get("audio_storage_path"),
    file_hash=input_hash,   # ← kwarg INEXISTENTE
)
```

Mas a função em `backend/storage/audit_storage.py:159` tem assinatura:
```python
def resolve_stored_audit_audio_path(relative_path: str | None):
```

Sem `file_hash`. Resultado: `TypeError: resolve_stored_audit_audio_path() got an unexpected keyword argument 'file_hash'` → FastAPI traduz em **HTTP 500** → frontend mostra toast genérico.

A função namespaced `MediaFileAdapter(relative_path, file_hash=...)` chegou a ser escrita em paralelo, mas ficou só no branch `wip-backup-2026-05-27` (commit `a70d3c4c`) e nunca foi mergeada em main. Sobrou um caller "novo" + função "antiga" — refactor pela metade.

## Estratégia

Reverter o caller para a assinatura antiga (sem `file_hash`). O parâmetro nunca funcionou em produção, então removê-lo não regride nenhum comportamento — só para de crashar.

Auditorias sem áudio passam a retornar **HTTP 404** ("Arquivo de audio da auditoria nao encontrado") em vez de 500, que é o que `AuthenticatedAudioPlayer` espera ("Gravação não disponível no servidor.").

## Mudanças

### `backend/routers/audit.py`
- `_resolve_existing_audit_audio_path` volta a chamar `resolve_stored_audit_audio_path(media_record.get("audio_storage_path"))` sem `file_hash`.
- Bloco que computava `input_hash` localmente fica também removido (era input apenas do kwarg quebrado).

Antes (linhas 45-63):
```python
def _resolve_existing_audit_audio_path(audit_id, media_record, audit=None):
    if media_record is None or not media_record.get("audio_storage_path"):
        return None
    input_hash = (
        (media_record.get("input_hash") if isinstance(media_record, dict) else None)
        or (audit.get("input_hash") if isinstance(audit, dict) else None)
        or f"audit_{audit_id}"
    )
    audio_path = resolve_stored_audit_audio_path(
        media_record.get("audio_storage_path"),
        file_hash=input_hash,
    )
    if audio_path is not None and audio_path.exists():
        return audio_path
    return None
```

Depois:
```python
def _resolve_existing_audit_audio_path(audit_id, media_record, audit=None):
    if media_record is None or not media_record.get("audio_storage_path"):
        return None
    audio_path = resolve_stored_audit_audio_path(media_record.get("audio_storage_path"))
    if audio_path is not None and audio_path.exists():
        return audio_path
    return None
```

Os parâmetros `audit_id` e `audit` ficam preservados para não quebrar o caller (`_recover_audit_audio_from_classified_queue` ainda usa essa assinatura).
