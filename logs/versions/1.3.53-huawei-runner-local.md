## Objetivo

Dar um caminho pragmatico para o sync Huawei rodar em producao sem
depender de servidor 24/7 nem de whitelist de IP em nuvem. A estrategia
ficou: rodar numa maquina da rede NSTECH (IP ja whitelisted pela Teledata)
disparada pelo **Windows Task Scheduler** em horarios fixos. O codigo fica
pronto para ser reutilizado depois em GCloud (Cloud Run Jobs + Cloud NAT)
ou Azure (Functions + NAT Gateway) sem alteracao.

## Mudancas

### Backend

- `backend/scripts/run_huawei_sync.py` (novo): runner CLI standalone.
  - Carrega `backend/.env` via `python-dotenv` (fallback gracioso se nao
    estiver instalado).
  - Configura `sys.path` para resolver imports de `core.huawei_sync`.
  - Argumentos `--horas` e `--log-level`; defaults lidos de env.
  - Gera log por execucao em `backend/logs/huawei_sync/<YYYYMMDD_HHMMSS>.log`.
  - Imprime JSON do relatorio de sync no stdout (para Task Scheduler
    capturar na History).
  - Exit codes: 0 sucesso (inclui stub quando `ENABLE_HUAWEI_SYNC=false`),
    1 excecao fatal, 130 interrupcao por teclado.

### Scripts / Ops

- `scripts/huawei_sync.bat` (novo): wrapper Windows para Task Scheduler.
  Resolve automaticamente a venv em `backend\.venv`, troca para o
  diretorio `backend/` (equivalente ao `uvicorn` do dev) e repassa args.
  Retorna o exit code original para a Task agendada.

## Como validar

1. Preencher `backend/.env` com AK/SK/ccId/vdn + `ENABLE_HUAWEI_SYNC=true`.
2. Rodar manualmente a partir da maquina NSTECH:
   ```powershell
   cd C:\Users\<voce>\projetos\auditoria
   scripts\huawei_sync.bat --horas 2
   ```
3. Conferir `backend\logs\huawei_sync\<timestamp>.log` e a entrada na
   tabela `huawei_sync_logs`.
4. Se OK, criar a task no Task Scheduler conforme secao 4 do
   `GUIA_EXECUCAO_LOCAL_HUAWEI.md`.

## Limitacoes conhecidas

- Enquanto rodar em desktop, o agendamento depende da maquina estar
  ligada ou a task configurada com "Run whether user is logged on or
  not" + senha armazenada. Notebook desligado = sync nao executa.
- Idealmente, instalar em um desktop/VM da NSTECH que fique sempre
  ligado (ex: estacao do supervisor de auditoria).
