# v1.3.155 — Extrai trilha de auditoria de colaboradores de operators.py

**Data:** 2026-06-15
**Tipo:** Refatoração conservadora (sem mudança de comportamento)
**Custo:** nenhum
**Branch:** `main`
**Fase:** Plano de arquitetura — corte P1 (3/4)

`repositories/operators.py` misturava acesso a `colaboradores` com a trilha de
auditoria (`colaboradores_audit_log`). Esta extração formaliza uma dependência
que já existia: `repositories/admin_criteria.py` já importava estes helpers de
`operators`.

## O que mudou

- **NOVO** `repositories/colaboradores_audit_log.py`: `_AUDITABLE_FIELDS`,
  `_snapshot_colaborador`, `_log_colaborador_audit`. Imports próprios
  (`typing.Optional`, `psycopg2.extras.Json`).
- `operators.py` **reexporta** `_snapshot_colaborador` e `_log_colaborador_audit`
  (usados internamente pelas funções CRUD e importados por `admin_criteria`).
  Removido o `from psycopg2.extras import Json`, que só era usado pela função
  movida.

## Por que era seguro

- **Cluster coeso**: as 3 entidades só se referenciam entre si (`_snapshot`
  usa `_AUDITABLE_FIELDS`; ambas recebem `cursor` por parâmetro). `_AUDITABLE_FIELDS`
  não tem nenhum outro uso em `operators.py`.
- **Sem monkeypatch**: nenhum teste faz patch em
  `repositories.operators._snapshot_colaborador`/`_log_colaborador_audit` (grep vazio).
- **Caminhos de import preservados**: callers internos (create/update/delete_colaborador)
  usam os nomes nus → resolvidos pelo reexport; `admin_criteria` faz
  `from repositories.operators import _snapshot_colaborador, _log_colaborador_audit`
  → resolvido pelo reexport.
- **Sem ciclo**: o novo módulo só importa `typing` e `psycopg2.extras`.

## Prova de equivalência

- Extração por **cópia byte-a-byte** (preserva o SQL) → `ast.dump`: as 2 funções
  e o assignment `_AUDITABLE_FIELDS` **idênticos** HEAD vs novo módulo.
- Reexport por identidade: `operators._snapshot_colaborador is colaboradores_audit_log._snapshot_colaborador`
  (idem `_log_colaborador_audit`); import-path do `admin_criteria` resolve.
- `Json` removido do namespace de `operators`; `_AUDITABLE_FIELDS` (16 campos) no novo módulo.

## Validação

- Suite backend completa (banco de teste local; nunca prod): verde.

`operators.py`: 1309 → 1243 linhas. Último corte P1: `classified_audio_storage`.
