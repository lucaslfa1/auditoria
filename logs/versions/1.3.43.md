## Objetivo
- Estabilizar a integração da IA na nuvem (Azure OpenAI), corrigir o erro de deploy relacionado a injeção falha de chaves de ambiente, e criar uma "blindagem de UX" para falhas do provedor.

## Bugs Corrigidos
- **BUG-001: Erro Internal 500 no endpoint de Auditoria** - Causado pela junção em cascata na variável `AZURE_OPENAI_ENDPOINT` devido ao PowerShell ter parseado `,` erroneamente no CLI do Google Cloud durante deploy. Resolvido através de `cmd /c "gcloud... "` re-separando adequadamente as chaves de Endpoint, Key e Deployment (`gpt-4o`).
- **BUG-002: Falha Silenciosa de IA Propagando 500 Global** - Quando as credenciais falhavam ou o modelo dava erro 404, o servidor FastAPI subia uma exceção de runtime nativa crasheando a call.

## Mudanças
- **Dashboard e Performance:** Componente visual do Painel de Performance validado em produção (re-cacheado por novo build de Vite).
- **Tratamento de Exceções de IA:** Roteador `/api/audit` agora levanta de forma graciosa um `HTTP status 503 (Serviço Indisponível)` em vez de `500` quando o provedor Azure falha.

## Conclusões Técnicas
- Ao trabalhar no terminal powershell windows injetando listas separadas por virgulas no `--update-env-vars`, a CLI do google cloud recebe array invés de string. Usar aspas rígidas (`""`) ou invólucro no `cmd.exe` evita horas de debugger.
- Deletamos a abordagem de fallback para CI/CD porque a diretriz nova define o ambiente 100% focado no serviço da Azure de LLM.
