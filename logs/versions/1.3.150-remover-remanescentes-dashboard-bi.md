# v1.3.150 — Remove remanescentes do módulo Dashboard/BI in-app

**Data:** 2026-06-15
**Tipo:** Remoção de código morto (sem mudança de comportamento)
**Custo:** nenhum
**Branch:** `main`

O módulo de Dashboard/BI in-app (telas "Dashboard de Qualidade" e "Performance",
componentes `Dashboard.tsx`/`PerformanceDashboard.tsx`) foi removido em
v1.3.146–149. Restavam fragmentos mortos espalhados que faziam parecer que o
módulo "não tinha sumido de verdade". Esta versão limpa todos eles.

> Não confundir com o que é núcleo e **fica**: `saveToDashboard`,
> `useAutomacaoDashboard`, os endpoints `/api/dashboard/*` de auditorias salvas
> (Arquivos Salvos), `/api/sectors`, e `repositories/analytics_quality.py`
> (indicadores do Portal Supervisor). O **Fechamento** (export para BI externo)
> também é intocado.

## O que mudou

### Backend
- `routers/system.py`: removido o endpoint morto `GET /api/analytics`
  (`get_audit_analytics`). Era consumido só pela tela in-app já removida; zero
  callers no frontend atual.
- `db/database.py`: removida a fachada `get_analytics`.
- `repositories/analytics.py`: removida a função `get_analytics` (~120 linhas).
  As demais funções do arquivo **ficam** (`get_stats`, `get_history`,
  `get_sectors`, `get_technical_incidents`) — alimentam `/api/dashboard/*` e
  `/api/sectors`.

### Frontend
- `shared/components/Sidebar.tsx` e `features/audit/hooks/useAuditOrchestrator.ts`:
  removido o valor morto `'dashboard'` das uniões `ViewType`.
- `index.css`: removido o bloco de estilos `.perf-*` (Performance Dashboard,
  ~375 linhas). Os overrides `.recharts-*` ficaram — `AuditScoreChart` (gráfico
  de nota da auditoria) ainda usa recharts.

### Testes
- `tests/backend/test_e2e_pipeline.py`: `/api/analytics` saiu da lista de
  prefixos verificados no startup (não há mais rota com esse prefixo).
- `tests/backend/test_audio_quality_persistence.py`: removida a chamada a
  `get_analytics` e suas asserções; o teste passa a exercitar só
  `get_stats`/`get_history`/`get_technical_incidents` (a classe segue marcada
  `@unittest.skip` desde a migração para PostgreSQL).

### Docs
- `docs/architecture/process-flow.md`: removidas as linhas das views
  `Dashboard`/`Performance` e a referência a `/api/analytics`.

## Por que era seguro

- Grep em `src/` por `/api/analytics`: **zero** chamadas no frontend atual.
- Grep em todo o repo por `get_analytics`/`get_audit_analytics`: os únicos
  callers eram o endpoint morto (via fachada) e um teste sempre skipado.
- O dump de rotas do app confirma: `/api/analytics` ausente; `/api/sectors` e
  todos os `/api/dashboard/*` presentes.

## Validação

- `python -m py_compile` nos arquivos backend alterados: OK.
- Import de `db.database`, `repositories.analytics`, `routers.system`: OK;
  `get_analytics` ausente; `get_stats`/`get_history`/`get_sectors` presentes.
- `npx tsc --noEmit` + `npm run build`: verde (bundle `AuditScoreChart` mantido,
  sem bundles de Dashboard/Performance).
- Suite backend completa (banco de teste local; nunca prod): verde.

## Resultado

Módulo Dashboard/BI in-app totalmente removido — sem endpoint, fachada, função
de repositório, tipos de view, CSS ou doc remanescentes.
