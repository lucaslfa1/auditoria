# v1.3.109 - Cortes de custo Azure: `fast` sem fallback premium por padrão

**Data:** 2026-06-10
**Tipo:** Custo / Transcrição
**Status:** implementado, testes-chave passam, commitado e enviado para `main` (deploy automático via Actions "Build and Deploy to Cloud Run")

## Contexto

Emergência de estouro de orçamento da API **Azure**. A investigação (relatório
completo em `~/Desktop/relatorio-consumo-azure-2026-06-10.md`) apontou o **ciclo
de automação recorrente** como driver — não os downloads nem a pré-triagem de
direção por áudio (que rodou 3×). A cada ~30 min o ciclo classificava com GPT-4o
e auditava com a engine mais cara (`hybrid_dual`), e **toda falha de `fast`
cascateava para Whisper + GPT-4o Diarize + SDK** (fallbacks forçados `True` no
código), multiplicado por retries (`max_retries=15`).

Resposta em 4 cortes: 3 de **infra** (já ao vivo, fora deste commit) + 1 de
**código** (este commit).

## Mudança

**`backend/core/transcription.py`** (o corte principal):
- Os fallbacks premium **deixam de ser forçados `True`**. Antes:
  `allow_whisper_fallback = True` / `allow_gpt4o_diarize_fallback = True`
  (hardcoded para "blindar o hybrid_dual"), e `AZURE_SDK_FALLBACK` default `True`.
- Agora são **condicionais**: só ligam quando a engine selecionada já é
  `hybrid_dual`/`whisper`/`gpt4o_diarize`, **ou** via flag explícita
  (`AZURE_PREMIUM_TRANSCRIPTION_FALLBACK`, `AZURE_WHISPER_FALLBACK`,
  `AZURE_GPT4O_DIARIZE_FALLBACK`, `AZURE_SDK_FALLBACK` — todas default `False`).
- **Efeito:** `fast` que falha **não cascateia mais** para as engines caras →
  vai para retry/triagem manual.

**`backend/core/automation_cache.py`** (sem mudança de lógica):
- `_satisfies_hybrid_dual_policy` → `_satisfies_transcription_policy`
- Motivo persistido `transcricao_hybrid_dual_falhou` → `transcricao_premium_falhou`
- Reason code `hybrid_dual_failed` → `premium_transcription_failed`
- O gate já aceitava **qualquer** estratégia válida (só rejeita transcrição
  ausente/vazia); a renomeação alinha nomenclatura ao novo padrão.

**`backend/.env.example`** — documenta `AZURE_TRANSCRIPTION_ENGINE=fast` e as 4
flags de fallback `=false`.

**`GEMINI.md`** — política "AI Quality Priority" (hybrid_dual como "motor oficial
e absoluto primário") substituída por "AI Cost/Transcription Policy" (`fast` é o
padrão; `hybrid_dual` é rota premium/opt-in).

## Comportamento alterado (registrar)

A cadeia documentada era `fast → whisper → gpt4o_diarize → sdk`. Agora, por
padrão, **`fast` não tem fallback automático**: numa falha do Azure Fast, o item
vai para **retry/triagem manual** em vez de gastar engine premium. **Reversível
sem deploy** via `AZURE_PREMIUM_TRANSCRIPTION_FALLBACK=true`. Pode **aumentar o
volume de triagem manual** se o Fast tiver soluços.
