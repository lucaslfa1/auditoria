# v1.3.178 — Limpeza de artefatos não usados (enxugar repo p/ handover)

**Data:** 2026-06-16
**Tipo:** Limpeza de repositório (chore)
**Custo:** nenhum
**Branch:** `main`

## Contexto

Para o handover, o repo deve conter apenas a documentação canônica e arquivos
realmente usados pelo sistema (runtime/build/testes). Uma auditoria classificou
os arquivos não-código em USADO / DOC / ARQUIVAR / APAGAR. Este commit remove o
primeiro lote — **lixo verificado** (sem uso em runtime/build/testes), que era a
maior parte do peso do repo (~81 MB → ~bem menor).

## Removido (verificado sem referência em código/config/testes)

- **`ligacoes/`** (~69 MB, 50 gravações de áudio de exemplo BOAS/RUINS/ZERADAS por
  setor). Não é lido em runtime; `.dockerignore`/`.gcloudignore` já a excluíam do
  build/deploy; o seeder manual `importar_ligacoes_auditadas.py` recebe a pasta por
  argumento (`--pasta`) e foi importação inicial one-time.
- **`docs/integracoes/huawei/reuniao_liberacao_obs.html` + `reuniao_liberacao_obs_files/`**
  (~6 MB): página web de terceiros salva (bundles JS do SharePoint/Office), sem
  valor de handover.
- **`backend/data/benchmark_diarization_*.json`** (11 arquivos): saídas stale de
  benchmark de diarização; nenhuma referência em código (só em scripts de benchmark).
- **`backend/data/pesos_gestores.json`**: cópia duplicada/obsoleta — o runtime usa
  `backend/core/pesos_gestores.json` (`core/export_gestores.py:26`).
- **`export/db_review/…json`**, **`export/relatorio_aplicacao.md`** (descrevia stack
  errada — desinformação), **`export/transcricao_teste_*.wav.pdf`** (saída one-off).
- **`planejamento/walkthrough.md.resolved`** (artefato stale com sufixo de merge),
  **`docs/reports/integracao-huawei-aicc-ciclo-sync.cjs`** (código stale solto em
  docs, não invocado por nada).

## Preservado (NÃO removido)
