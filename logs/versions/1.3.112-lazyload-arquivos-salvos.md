# v1.3.112 - Lazy-load dos Arquivos Salvos (corte de egress Neon)

**Data:** 2026-06-10
**Tipo:** Custo / Performance (Neon) / UX
**Status:** implementado, `tsc -b` limpo, commitado e enviado para `main` (deploy via Actions)
**Origem:** verificado e commitado durante a sessão.

## Contexto

A tela **Arquivos Salvos** carregava a lista com `SELECT *` — trazendo o conteúdo
inteiro de cada arquivo + metadata completo (e, em `ai_feedback`, até o vetor
`transcricao_embedding`). Em listas grandes isso é egress/transfer desnecessário
do Neon a cada abertura. Esta mudança troca para **lista enxuta + detalhe sob
demanda**.

## Mudança

**`backend/repositories/saved_files.py`** — `list_arquivos_salvos`:
- `SELECT *` → colunas explícitas.
- `conteudo` → **preview de 800 chars** (`LEFT(...)`) na listagem.
- `metadata_json` completo → **`metadata_summary_json`** (só summary, ai_feedback,
  score, source_type, timestamp, audio_date, operator_id) via `jsonb_build_object`.
- Cada item da lista ganha `detail_loaded: false`.
- `get_arquivo_salvo` (detalhe por id) devolve o registro completo + `audit_status`
  + `detail_loaded: true`.

**`backend/core/ai_feedback.py`** e **`backend/repositories/admin_criteria.py`** —
`SELECT *` → colunas explícitas (validadas contra o schema real). Em `ai_feedback`,
isso exclui o vetor `transcricao_embedding` da listagem/get.

**`src/features/saved-files/components/SavedFiles.tsx`** — lazy-load:
- Ao selecionar um item, mostra o preview e busca o detalhe completo em
  `GET /api/salvos/{id}` (`loadFullItem`), com **guarda de corrida**
  (`selectedRequestRef`) para cliques rápidos.
- Spinner "Carregando detalhes..." e botões (editar/exportar/excluir/enviar)
  desabilitados enquanto carrega.

## Validação

- Colunas dos `SELECT` explícitos **conferidas contra o schema real** (Neon
  `information_schema`): `ai_feedback`, `audit_sectors`, `audit_alerts`,
  `audit_criteria`, `arquivos_salvos` — todas existem.
- `npx tsc -b` → exit 0.
- Rota `GET /api/salvos/{arquivo_id}` existe (`backend/routers/saved_files.py:163`).
