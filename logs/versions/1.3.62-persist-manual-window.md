## Objetivo
- Persistir o intervalo manual (data/hora de inicio e fim) do painel de download de ligacoes da Huawei entre navegacoes/F5.

## Contexto
No modulo Telefonia, o painel "Download de ligacoes" tem tres campos de configuracao:
- modo de coleta (`retroativo` | `manual`)
- horas retroativas (quando `mode = retroativo`)
- data/hora de inicio e fim (quando `mode = manual`)

Antes desta versao, somente `mode` e `retroactiveHours` eram persistidos via `localStorage`. Os campos de intervalo manual (`beginAt`, `endAt`) eram reinicializados a cada montagem do componente para os defaults (`hoje 00:00` ate `agora`).

## Sintoma
Usuario configurava manualmente um intervalo (ex.: 03/05 08:00 ate 03/05 12:00), saia da pagina de telefonia ou dava F5, voltava e os campos haviam revertido para `hoje 00:00 → agora`. Se clicasse em "Puxar ligacoes agora" sem reconfigurar, a sincronizacao puxava o intervalo default, nao o que ele tinha definido.

## Causa Raiz
`src/features/telefonia/components/SyncPanel.tsx` linhas 120-121:
```typescript
const [beginAt, setBeginAt] = useState<string>(defaultBegin);
const [endAt, setEndAt] = useState<string>(defaultEnd);
```
`defaultBegin` retorna sempre `hoje 00:00`; `defaultEnd` retorna sempre `agora`. Nao havia leitura nem escrita em `localStorage` para esses dois states.

## Mudancas

### Frontend
- **`src/features/telefonia/components/SyncPanel.tsx`**:
  - Adicionadas duas novas chaves de storage:
    - `STORAGE_BEGIN_AT_KEY = 'telefonia_manual_begin_at'`
    - `STORAGE_END_AT_KEY = 'telefonia_manual_end_at'`
  - Inicializacao de `beginAt` e `endAt` passou a ler `localStorage` primeiro, caindo no default apenas quando ausente.
  - Adicionados dois `useEffect` que escrevem `beginAt`/`endAt` em `localStorage` a cada mudanca (mesmo padrao ja usado para `mode` e `retroactiveHours`).

## Comportamento Pos-Fix
- F5 na pagina de telefonia → mantem o intervalo manual configurado.
- Navegacao para outro modulo e retorno → mantem o intervalo.
- Primeira visita ou storage limpo → continua usando `hoje 00:00 → agora`.

## Riscos
- Baixo: alteracao isolada ao componente do painel de download. Nao toca em logica de backend, prompt da IA, classificacao ou banco.
- Edge case: se o usuario nunca tocar nos campos manuais e o storage ainda guardar valores muito antigos (ex.: de varios dias atras), eles permanecerao ate ele alterar. Isso e o comportamento desejado (persistencia da ultima escolha do usuario), mas vale documentar para suporte: se a sincronizacao trouxer "zero ligacoes", verificar se o intervalo manual nao esta apontando para uma data antiga.
